# Limitations & Observations

## Objective
This section documents challenges encountered during the implementation of alerting and monitoring in the Azure environment.

---

## 1. Alert Triggering Inconsistency

Although the alert rule for suspicious SSH activity was successfully created and configured, alerts were not consistently triggered during testing.

### Observed Behavior
- Suspicious SSH activity was visible in Log Analytics queries
- Detection queries returned valid results (e.g., "Invalid user")
- Alert rule was properly configured with an action group
- However, no alerts were triggered or visible in the Alerts panel

---

## 2. Root Cause Analysis

The issue was traced to the Data Collection Rule (DCR) configuration.

### Key Finding
The active Data Collection Rule did not include **Linux Syslog** as a configured data source.

### Impact
- SSH-related logs (auth, authpriv) were not consistently ingested
- Alert rule queries depended on Syslog data
- Inconsistent or missing data prevented reliable alert triggering

---

## 3. Additional Configuration Challenge

During attempts to fix the issue:

- Azure returned an error related to duplicate destination names:

---


### Cause
- Attempting to create multiple destinations within the same Data Collection Rule
- Azure internally assigned duplicate names (e.g., `vminsightworkspace`)

### Resolution Approach
- Identified that the existing Log Analytics workspace should be reused
- Avoided creating duplicate destinations
- Attempted to reconfigure Syslog using the existing workspace

---

## 4. Platform Behavior Considerations

Azure monitoring and alerting behavior can be affected by:

- Log ingestion delay
- Data Collection Rule configuration
- Evaluation frequency of alert rules
- Azure for Students subscription limitations

---

## 5. Key Learning
Detection logic is only effective if the data pipeline is correctly configured.

Even if:
- queries are correct
- alerts are configured

Missing or inconsistent log ingestion will break detection.

---

## 6. Security Insight

Cloud security monitoring requires multiple layers to function correctly:

- Data Collection (DCR)
- Log Storage (Log Analytics)
- Detection Logic (KQL queries)
- Alerting (Alert Rules + Action Groups)

Failure in any one layer can impact the entire detection pipeline.

---

## Outcome

Although alert triggering was not fully validated due to configuration constraints, the project successfully demonstrated:

- log collection setup
- detection query development
- alert rule configuration
- identification and analysis of monitoring pipeline issues

This reflects realistic challenges faced in cloud security environments.
